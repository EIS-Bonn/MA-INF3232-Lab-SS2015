Ontologies at a glance

The Web-based partition and ontology tool (WAPT) serves as a tool for ontology engineers with dedicated usage. Its main scope is to help analysis and ontology reuse.

About

The scope of the WAPT project is defined in what follows :

  • analyze aspects like structure and semantics of vocabularies or ontologies
  • understand the content and evolution of vocabularies or ontologies
  • support the partitioning through existing modularization techniques
  • employ the MEAN stack with a NodeJS backend as its software development base
  • export ontologies
  • work cross platform

The application flow

It is very important that the steps are followed in order!

  1. Upload

    2. The first step into the application is to upload an ontology. There are two options of uploading:

    • From local computer
      A OWL file can be uploaded.
    • From the internet
      An ontology can be uploaded from an existing URL on the Internet. For example:

    Choosing either of the options the ontology is parsed using specific tools. The error messages are marked with red color, while the warning messages (which are not critical for the running of the application) are marked with orange. Success messages are marked with green.

  2. Parse

    3. Parsing is the second step of an ontology analysis. In this step, metrics are displyed. These metrics include number of classes, axioms, properties, blank nodes. By vieweing the results in a table, the ontology engineer can have at a quick glance a quick statistic of the ontology.

  3. Visualize

    4. After having parsed the ontology, another step in analyzing the ontology can be taken. This time, the ontology engineer can visualize the ontology. On the right hand side, a graph is displayed. This graph has all the triples of the ontology. Furthermore, a list of checkboxes can be seen on the left hand side. Here, certain elements can be highlighted. There are two types of highlights possible, either by color, or by shape. A highlight by shape is done such that a color is not overriden.

  4. Partition

    5. The partition is one of the vital parts of the tool. It allows partitiong the ontology into a specified number of graphs.

    The user can specify the number of graphs by dragging the slider.

  5. Export

    6. Finally, the ontology can be exported, after clicking on the link for exporting. A text file with triples will be exported.
